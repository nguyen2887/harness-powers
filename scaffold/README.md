## Development Model

This repository uses the Harness operating model for AI-assisted development:
work enters through feature intake, is classified into a risk lane, tracked as
stories, and proven through the durable layer.

- Operating model: `docs/HARNESS.md`
- Work classification: `docs/FEATURE_INTAKE.md`
- Product truth: `docs/product/`
- Stories: `docs/stories/`
- Decisions: `docs/decisions/`
- Proof status: `scripts/bin/harness-cli query matrix`
  (`.\scripts\bin\harness-cli.exe` on Windows)
